# 更新日志 - 2026年4月4日

## 文档重构

### 1) 重写存储扩容总览

- 更新 `docs/Storage_Expansion_Guide.md`
- 更新 `docs/en/Storage_Expansion_Guide.md`
- 将主线调整为：
  - 先判断当前是 `ext4` 还是 `squashfs`
  - 再在“根分区扩容 / 数据分区 / extroot”之间选路线
- 补充树莓派 `ext4` 根分区扩容流程，包括：
  - 备份
  - 清理残留 `fstab.overlay`
  - 使用 `parted` 检查分区
  - 离线扩容建议
  - 使用 OpenWrt 官方脚本在线扩容
  - 扩容成功后的验证与后续操作建议

### 2) 收紧 extroot 文档适用范围

- 更新 `docs/ExtendOverlaySize.md`
- 更新 `docs/en/ExtendOverlaySize.md`
- 明确说明这篇只适用于：
  - `squashfs + overlay`
  - `/overlay` 软件空间不足
- 增加显式提示：
  - 如果系统是 `ext4` 根分区，不要照着 extroot 教程操作
  - `extroot` 的关键是迁移旧 overlay，而不是单纯挂载新分区

### 3) 清理维护文档中的过时分区描述

- 更新 `docs/System_Maintenance.md`
- 更新 `docs/en/System_Maintenance.md`
- 移除旧的固定 `p3 / overlay / 10GB` 示例，避免与实际镜像布局冲突
- 将存储相关内容改为：
  - 先检查 `mount`、`df -h`、`block info`、`uci show fstab`
  - 再跳转到主扩容文档
- 同步修正智能备份说明，统一为当前策略：
  - 仅在配置变化时生成备份
  - 本地保留 3 份
  - 云端保留 7 份

### 4) 增加“避免重复内容”的文档规范

- 更新 `docs/Documentation_Style_Guide.md`
- 更新 `docs/en/Documentation_Style_Guide.md`
- 新增规则：
  - 同一主题只能有一个主教程
  - 其他文档只负责适用范围、风险说明和跳转
  - 避免在多个文档重复粘贴同一套操作步骤

### 5) 调整 README 扩容入口文案

- 更新 `README.md`
- 更新 `README_EN.md`
- 将扩容入口描述改为：
  - 先判断系统类型，再选路线
  - extroot 仅适用于 `squashfs + overlay`

## 结果

- 读者先判断系统类型，再执行扩容命令，误操作风险更低
- 树莓派 `ext4` 固件用户可以直接找到正确路线
- `extroot` 不再被误当成通用方案
- 扩容、维护、README 之间的职责边界更清晰，重复内容更少
